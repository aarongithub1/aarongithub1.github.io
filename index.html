When would you use ‘=’ vs ‘==’ vs ‘.equals’? Explain what each of these does and in which cases you should use them.

In Java, the ‘=‘ is used to direct data to be stored into a variable. The data to the right of the ‘=‘ will be stored into the variable to the left of the ‘=‘.

Some examples of how this works:

int Num = 0;
(int Num is “defined”, variable was created and assigned the value of 0 at the same time.)
————————————-
int Num;
Num = 5;
(Num will store the value of 5)
————————————-
int Num;
Num = 5*5;
(Num will store the value of 25)
————————————-
int Num1, Num2, Num3;
Num1 = 10;
Num2 = 20;
Num3 = Num1+Num2;
(Num3 will store the value of 30, it will not store the formula - "Num1+Num2")

The ‘==‘ is one of six Boolean operators for comparing items.

> (greater than)
>= (greater than or equal to)
< (less than)
<= (less than or equal to)
== (equal to)
!= (not equal to)

If the items being compared elavuate to a "true" statement then the value "true" will be stored into the variable.

For example:

int x = 5, y = 10;
boolean a, b, c;

a = (x < y);
b = (x == y);
c = (x != y);

The first statement will result in a "true" being stored into 'a', because x is less than y.
The second statement will result in a "false" being stored into 'b'
, because x is not equal to y.
And the third statement will result in a "true" being stored into 'c', because x is not equal to y.

Finally, if you are trying to compare two 'strings' rather than 'integers/doubles' you will need to use the built-in method '.equals()'

For example:

String word = "hello";
boolean yes, no;

yes = word.equals("hello");
(This is true)

no = ! word.equals("hello");
(This is false)

yes = ! word.equals("testing");
(This is true)

no = word.equals("testing");
(This is false)

What is "void" in a method declaration? How does this differ from declaring a method with a return type?

A "void" in a method declaration indicates that the method will not return a value. If you want to return a value from a method the "return type" must be indicated in the method declaration.

For example:

public static void testMethod(int testInt) {

}

The example above will accept one integer, received from the Method Call, and declares a local variable called "testInt". After running any code between the curly braces, it will return nothing.

public static double randomMethod(int x, int y) {
double tempDouble;

tempDouble = x / y;

return tempDouble;
}

or simply:

public static double randomMethod(int x, int y) {

return x / y;
}

In these examples, "randomMethod" will accept two integers and declare int x and y as local variables.
After it has completed it's task, it will return a "double" value.

Note: (You may have learned this as a Function, but in Java, they have Methods rather than Functions.)

blog 3

What happens when you include ‘import java.util.Scanner;’ in your code? Where does this functionality come from? What is the ‘java’ in ‘java.util.Scanner;’?

The line of code “import java.util.Scanner;” is used to create a Scanner object for accepting user input from the Standard input (System.in). This functionality comes from the Scanner class within the java.util.* package.

Java has a very large class library (a grouping of packages) called the Java API, the acronym for “Application Programming Interface.” The "java" in "java.util.Scanner;" can be thought of as a file folder, or a way to organize related classes within the Java API, where "java" is the main folder containing all the classes and functionality a developer might need.

Note: (There are other classes within the “java.util.*” package besides the Scanner class.)

What does it mean to ‘cast a double to an int’? Give an example of when you would do this and explain what is happening.

When we have seen casting, it has been used with the built-in Math.random() method. It does not require the Math.random() method to perform 'casting.'

Basicly what casting is doing, is taking a fractional number and shaving off everything to the right of the decimal. (turning it into a whole number)

So, in the example:

i = (int)(9.98765);
System.out.println("i = "+i);
output will be: 9

This is because 9.98765 will have everything to the right of the decimal removed, and will only accept the whole number 9 when it is cast into an integer.
